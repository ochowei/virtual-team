# GOALS.md 模板與 AI 生成提示詞

本文件提供兩種方式來建立新專案的 `GOALS.md`：

- **方式 A**：直接複製下方骨架，手動填寫
- **方式 B**：複製 AI 生成提示詞，貼到 Claude / Gemini / ChatGPT，讓 AI 幫你生成

建立好後，將產出的內容存為 `.virtual-team/GOALS.md`。

---

## 方式 A：GOALS.md 結構骨架

將以下內容複製並替換 `[佔位符]` 為你的專案內容：

```markdown
# [專案名稱] — 開發目標清單

**最後更新：** [YYYY-MM-DD]
**負責人：** Human Admin + AI Virtual Team

---

## MVP 範圍 (Phase 1)

Phase 1 專注於以下任務（達成後即為 MVP 可交付）：

- Task [X.X.X]：[任務名稱]
- Task [X.X.X]：[任務名稱]

**MVP 驗收標準：**
- [ ] [可被測試驗證的行為描述]
- [ ] [可被測試驗證的行為描述]

---

## Epic 1：[Epic 名稱]

### Feature 1.1：[Feature 名稱]

- [ ] `[TODO]` Task 1.1.1：[任務名稱]
  - 目標：[清楚說明這個任務的目的]
  - 產出：[完成後存在的檔案或行為]

- [ ] `[TODO]` Task 1.1.2：[任務名稱]
  - 目標：[清楚說明這個任務的目的]
  - 產出：[完成後存在的檔案或行為]

### Feature 1.2：[Feature 名稱]

- [ ] `[TODO]` Task 1.2.1：[任務名稱]
  - 目標：[清楚說明這個任務的目的]
  - 產出：[完成後存在的檔案或行為]

---

## Epic 2：[Epic 名稱]

### Feature 2.1：[Feature 名稱]

- [ ] `[TODO]` Task 2.1.1：[任務名稱]
  - 目標：[清楚說明這個任務的目的]
  - 產出：[完成後存在的檔案或行為]
```

## 方式 B：AI 生成提示詞

將以下提示詞複製，貼到 Claude、Gemini、ChatGPT 或其他 AI 工具。
**將 `[YOUR PROJECT DESCRIPTION]` 替換成你的專案描述**，然後送出。

---

```
你是一位軟體專案規劃師。請根據以下專案描述，幫我生成一份 GOALS.md 任務清單。

【專案描述】
[YOUR PROJECT DESCRIPTION]

【輸出格式要求】

請嚴格遵守以下格式：

---
# [專案名稱] — 開發目標清單

**最後更新：** [今天日期]
**負責人：** Human Admin + AI Virtual Team

---

## MVP 範圍 (Phase 1)

Phase 1 專注於以下任務（達成後即為 MVP 可交付）：

- Task [X.X.X]：[任務名稱]
（列出 Phase 1 的所有任務 ID 與名稱）

**MVP 驗收標準：**
- [ ] [具體、可被觀察或測試的行為]
（至少 3 條標準）

---

## Epic 1：[名稱]

### Feature 1.1：[名稱]

- [ ] `[TODO]` Task 1.1.1：[任務名稱]
  - 目標：[一句話說明為什麼要做]
  - 產出：[完成後會存在的檔案路徑或可觀察的行為]

（依此格式繼續列出所有 Feature 與 Task）
---

【規劃規則】
1. Phase 1 (MVP) 只包含 2–4 個 Epic，規模要在數週內可完成
2. 每個 Feature 包含 3–7 個 Task
3. 所有 Task 狀態一律為 `[TODO]`（未開始）
4. Task 命名必須是具體動作，例如「建立 GameState 類別」而非「設計遊戲」
5. 每個 Task 的「產出」必須是可驗證的（檔案存在 / 指令可跑通 / 功能可操作）
6. Epic 之間應有合理的依賴順序（例如「基礎建設」先於「核心功能」）
7. 不要加入 Phase 1 以外的 Epic，除非明確標示為「Phase 2（未來規劃）」

請直接輸出 GOALS.md 的完整內容，不要前言或解釋。
```

---

## 使用說明

1. 取得 AI 生成的內容後，檢查並調整：
   - Task 粒度是否合適（太大 → 拆分；太細 → 合併）
   - 依賴順序是否正確（後面的 Task 不應依賴尚未完成的前置 Task）
   - MVP 範圍是否合理（2-4 週能完成？）

2. 將確認後的內容存為 `.virtual-team/GOALS.md`（覆蓋本模板檔案旁的 GOALS.md）

3. 執行虛擬團隊前，確認所有 Task 狀態均為 `[TODO]`
